## Step 1: Search for Opacity

| Boundary | What Performance Data is Erased |
|----------|--------------------------------|
| **CLI subprocess boundary** (`ClaudeInterface.call()`, `ClaudeBackend.send()`) | API latency breakdown (DNS, TCP, TLS, processing), connection reuse, retry state, backpressure signals, streaming buffer pressure, internal throttling state |
| **File I/O boundary** (`.deep/` directory ops) | Disk I/O patterns (sequential vs random), cache hits/misses, file system journaling overhead, inode metadata costs, directory lookup performance |
| **Thread lock boundary** (`_findings_lock`, `_session_lock`, `_file_io_lock`) | Lock contention wait times, which threads hold locks longest, convoy effects, false sharing on cache lines, NUMA node migration |
| **Stream parsing boundary** (`StreamParser.parse_line()`) | JSON parsing overhead per line, memory allocation patterns during parsing, branch prediction on event type dispatch, GC pressure from event object creation |
| **Session resume boundary** (`--resume session_id`) | Token counting accuracy, context window utilization, cache hit rate for resumed sessions, deserialization cost of session state |
| **Plugin loading boundary** (`_load_prism()` with file paths) | File system cache performance, metadata parsing overhead, YAML frontmatter parsing cost, validation predicate costs |

## Step 2: Trace the Blind Workarounds

| Boundary | Blocked Optimization | Blind Workaround | Concrete Cost |
|----------|---------------------|------------------|---------------|
| **CLI subprocess** | Connection pooling (reuse HTTP/2) | Spawns new `subprocess.Popen` per call, temp file per system prompt | ~200ms process spawn + ~50ms temp file write per call; no connection reuse |
| **CLI subprocess** | Bidirectional streaming with backpressure | Fire-and-forget stdin write → read stdout | 120s hard-coded timeout; if CLI blocks, caller hangs |
| **CLI subprocess** | Transparent rate limit handling | Parses `rate_limit_event` but only logs it | Caller cannot back off proactively; hits limits anyway |
| **File I/O** | Batched writes (writev, scatter/gather) | Individual `pathlib.Path.write_text()` calls | ~5-10ms fsync per file; ~20 files × 10ms = 200ms per scan operation |
| **Thread locks** | Lock-free data structures (atomic counters) | `threading.Lock()` around all_results dict access | Lock acquisition ~1µs contended; 5 threads × 20 locks = 100µs overhead per batch |
| **Thread locks** | Per-file sharding (no lock coordination needed) | Single global lock for findings writes | Review mode serializes all writes; 5 files could write in parallel |
| **Stream parsing** | Zero-copy JSON parsing | `json.loads()` per line, allocates new dict | ~2µs JSON decode + ~1µs dict allocation per line; ~1000 lines = 3ms |
| **Stream parsing** | Event object pooling | Creates new tuple/list per event type | ~1000 events × 80 bytes = 80KB allocation per stream |
| **Session resume** | Incremental context updates | Full context re-send on resume | Token count: ~0.30 per input token; $0.30 per million vs $0.15 for incremental |
| **Plugin loading** | In-memory caching with reload detection | `pathlib.Path.read_text()` every `_load_prism()` call | ~2ms file read per prism; 6 prisms × 2ms = 12ms per pipeline |
| **Plugin loading** | Compiled metadata validation | YAML parsing via regex in Python every load | ~0.5ms frontmatter strip + ~0.3ms key:value parse per prism load |

## Step 3: Name the Conservation Law

| Boundary | Erased Data | Blocked Optimization | Blind Workaround | Concrete Cost | Flattening Breaks |
|----------|-------------|---------------------|------------------|---------------|-------------------|
| **CLI subprocess boundary** | API latency breakdown, connection reuse, retry state, streaming backpressure | Connection pooling (HTTP/2 reuse), bidirectional streaming with backpressure, transparent rate limit handling | Spawn subprocess per call, temp file for system prompt, 120s timeout | ~250ms spawn + temp file overhead per call; no connection reuse; hangs on CLI block | **Portability via CLI abstraction** — flattening requires direct Python SDK (anthropic package), breaking: cross-platform CLI compatibility, session resume via CLI state, ability to swap CLI implementations without code changes |

**The Conservation Law:** `Execution Transparency × Portability = constant`

- **Execution Transparency**: Direct API access gives latency visibility, connection pooling, bidirectional streaming
- **Portability**: CLI wrapper provides cross-platform compatibility and easy swapping of Anthropic's interface
- **Trade-off**: Gaining CLI portability (works on any system with `claude` installed) sacrifices all API-level performance data and control

**Secondary Conservation Laws:**

1. **Thread Safety × Throughput**: `threading.Lock()` prevents data races but blocks parallel writes (cost: ~100µs contended lock overhead per batch)

2. **File System Persistence × Latency**: Writing findings to disk provides durability but costs ~5-10ms fsync per file (total: ~200ms per scan for 20 files)

3. **Dynamic Plugin Loading × Startup Cost**: Runtime prism loading enables hot-reload (`/reload`) but costs ~2ms per prism load per pipeline
