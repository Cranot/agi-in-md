# L12G Structural Analysis: Starlette routing.py

## PHASE 1 — STRUCTURAL ANALYSIS

### Three Properties This Code Claims

1. **Composable route hierarchy** — Routes and Mounts can be nested arbitrarily via `Router.routes` list
2. **Clean parameter extraction** — Each route's `matches()` returns isolated `path_params` in `child_scope`
3. **Deterministic resolution** — Single pass through routes yields exactly one handler

### Proof They Cannot Coexist

The conflict centers on **parameter namespace collision across nesting boundaries**:

In `Mount.matches()`:
```python
path_params = dict(scope.get("path_params", {}))
path_params.update(matched_params)  # MERGES parent params into child
```

In `Route.matches()`:
```python
path_params = dict(scope.get("path_params", {}))  
path_params.update(matched_params)  # SAME merge pattern
```

**The impossibility**: If a Mount extracts parameter `id` from `/users/{id}` and delegates to a child Route that also extracts `id` from `/posts/{id}`, the child's `update()` **overwrites** the parent's value. The code claims clean isolation (each route extracts "its" params) while implementing inheritance via mutation (update merges unconditionally).

Proof by construction:
```python
# Mount at /users/{id} containing Route at /posts/{id}
# Request: /users/alice/posts/bob
# Mount extracts: id="alice", passes to child
# Route extracts: id="bob", UPDATES scope
# Result: id="bob" — parent's "alice" is lost
```

### Conservation Law

**Parameter uniqueness × Route composability = constant**

- To compose routes freely: parameters must be globally unique or namespaced
- To allow duplicate parameter names: routes must be isolated (no inheritance)
- The code chooses inheritance-by-update, sacrificing uniqueness silently

### Concealment Mechanism

The `dict.update()` pattern conceals the collision. It appears to "enrich" the scope with new parameters, but actually performs **destructive overwriting**. The line:

```python
path_params = dict(scope.get("path_params", {}))
```

Creates a shallow copy of the dict, suggesting isolation — but the copy happens **before** the destructive update, not after. The illusion: each route gets "its own" params. The reality: later routes overwrite earlier routes' params silently.

### Improvement That Recreates the Problem Deeper

```python
class NamespacedParams:
    def __init__(self, route_name, params):
        self.route_name = route_name
        self.params = params

# In matches():
child_scope = {
    "path_params": NamespacedParams(self.name, matched_params),
    "parent_params": scope.get("path_params")  # Preserve chain
}
```

This makes the hierarchy explicit. But it recreates the problem: `url_path_for()` now needs to traverse the parent chain to reconstruct URLs, and `replace_params()` must know which level's params to use. The conservation law holds: we've traded **silent collision** for **explicit traversal complexity**.

## Final Output

**Conservation Law (Corrected)**:
The code exhibits **parameter namespace collision under composition**. When routes are nested, parameter keys are shared via `update()`, making later extractions overwrite earlier ones. This is not a "law of nature" but a **structural property of the chosen merge strategy**.

**Verified Structural Findings**:

1. **Silent parameter overwrite** — `path_params.update()` in both `Route.matches()` and `Mount.matches()` destroys parent-level parameters when child uses same key
2. **Mutation as interface** — `replace_params()` modifies its `path_params` argument via `.pop()`, requiring callers to understand this side effect
3. **Ordering-dependent PARTIAL resolution** — Router remembers first PARTIAL match only; route ordering affects 405 response generation
4. **Mount path parameter dual-use** — `path` parameter serves as both capture group (matching) and synthetic placeholder (URL building), requiring special-case handling in `url_path_for()`
5. **Dead code in request_response** — Inner `async def app` shadows outer, making outer wrapper unreachable
