# Winning Hackathon Strategy for Hermes Agent

## Key Alignment Points

The "grows with you" motto suggests they value:
- **Personalization** that improves over time
- **Persistent learning** from user interactions
- **Adaptive behavior** based on user patterns

Prism's strength (cheap models doing structural analysis) means you should build something that:
- Analyzes patterns/structure efficiently
- Doesn't require expensive model calls
- Benefits from repeated, iterative analysis

---

## Top 3 Build Ideas

### 1. **Adaptive Code Mentor** (Highest Win Potential)
A skill that learns your coding patterns, common mistakes, and preferences—then proactively offers personalized suggestions.

```
Prism analyzes: commit patterns, bug frequencies, code review feedback
Learns: "This user often forgets error handling in async functions"
Grows: Accuracy improves with each interaction
```

**Why it wins:** Directly embodies "grows with you" + showcases Prism's pattern analysis

### 2. **Project Context Builder**
Automatically builds and maintains a living mental model of any codebase—growing deeper understanding over time without expensive re-analysis.

```
Uses Prism to: Extract structure, map dependencies, identify patterns
Output: A queryable knowledge base that grows with the project
```

### 3. **Personal Debugging Detective**
Learns from your past debugging sessions to predict and surface relevant context when new issues arise.

---

## Recommended Approach

Build **#1 (Adaptive Code Mentor)** because:
- Clear demo narrative ("watch it learn about me")
- Uses Prism's structural analysis on code/commit patterns
- Measurable "before vs after" improvement
- Practical value judges can immediately grasp
