# U5: Dual Process Theory + Implicit Bayesian Inference — Literature Review

**Date:** Mar 17, 2026
**Question:** Is L7→L8 a System 2→System 1 mode shift? Are prisms Bayesian prior specifications?

## Part 1: Dual Process Theory Applied to LLMs

### Core Papers

1. **Kahneman (2011)** — *Thinking, Fast and Slow*. System 1: fast, automatic, intuitive. System 2: slow, deliberate, analytical. System 2 requires working memory and attention allocation.

2. **Evans (2008)** — "Dual-Processing Accounts of Reasoning, Judgment, and Social Cognition." Dual-process theories distinguish Type 1 (autonomous, fast, high capacity) from Type 2 (controlled, slow, limited capacity).

3. **Nature Reviews Psychology (2025)** — "Dual-process theory and decision-making in large language models." LLMs exhibit BOTH System 1-like responses (cognitive biases, heuristics) AND System 2-like responses (careful reasoning) depending on prompting method. Chain-of-Thought shifts LLMs from System 1 to System 2. [nature.com](https://www.nature.com/articles/s44159-025-00506-1)

4. **Dualformer (2024)** — "Controllable Fast and Slow Thinking by Learning with Randomized Reasoning Traces." Explicitly trains transformer with both fast (System 1) and slow (System 2) reasoning traces. Shows System 2 can be activated by prompting. [arXiv:2410.09918](https://arxiv.org/html/2410.09918v2)

5. **Frontiers in Cognition (2024)** — "Dual-process theories of thought as potential architectures for developing neuro-symbolic AI models." Proposes using dual-process theory to design AI architectures.

### Mapping to Project Findings

| Feature | System 1 | System 2 | Project Analogy |
|---------|----------|----------|-----------------|
| Speed | Fast | Slow | L8 = immediate execution. L7 = requires meta-monitoring |
| Capacity | Unlimited | Limited | L8 works on Haiku (any capacity). L7 needs Sonnet minimum |
| Mode | Pattern completion | Deliberate evaluation | L8 = construction (procedural). L7 = meta-analysis (evaluative) |
| Trigger | Automatic | Effortful | L8 = triggered by imperative format. L7 = requires explicit meta-instruction |
| Error type | Bias/heuristic | Correct but slow | L8 = template-generated patterns. L7 = accurate but capacity-gated |

**Key insight from Nature 2025 paper:** CoT prompting shifts LLMs from System 1 → System 2. The project's finding is the REVERSE: construction-based prompting shifts from System 2 → System 1-compatible operations, BYPASSING the capacity gate. This is a novel contribution — most research tries to activate System 2. The project found that System 1-compatible operations can produce DEEPER outputs than System 2.

### Testable Predictions

1. **L7 rescue via System 2 scaffolding:** If L7 failure is wrong-mode, then "Think step by step about whether your claim is valid before proceeding" should rescue L7 on Haiku. This is literally Chain-of-Thought applied to the meta-analytical step.

2. **System 1 depth paradox:** If L8 is System 1 and produces deeper analysis than L7 (System 2), this challenges the dual-process assumption that System 2 = better. The project may have found a domain where System 1-compatible operations are structurally superior — not because they're faster, but because they BYPASS the meta-analytical bottleneck.

3. **Vocabulary as mode switch:** "Code nouns" may activate System 1 analytical mode (familiar, automatic pattern completion). "Abstract nouns" may activate System 2 (unfamiliar, requires deliberate interpretation). This is testable with the vocabulary experiments in Rounds 32-34.

### Limitations

- LLM "reasoning" is not the same as human dual-process cognition (Nature 2025)
- LLM "cognitive biases" reflect training data patterns, not evolved heuristics
- The System 1/2 metaphor may be misleading — LLMs don't have two systems, just different generation regimes activated by context

## Part 2: Implicit Bayesian Inference (Xie et al. 2021)

### Core Paper

**Xie et al. (2021)** — "An Explanation of In-context Learning as Implicit Bayesian Inference" NeurIPS. [arXiv:2111.02080](https://arxiv.org/abs/2111.02080)

**Key claim:** When the pretraining distribution is a mixture of latent "concepts" (HMMs), in-context learning emerges as the model implicitly performing Bayesian inference in its forward pass. The model marginalizes over hidden concepts to predict the next token.

**Mechanism:** Documents are generated by: (1) sample concept θ from prior p(θ), (2) generate tokens from p(tokens|θ). The model learns p(y|x) by marginalizing: p(y|x) = Σ_θ p(y|θ)·p(θ|x). The "prompt" acts as evidence that collapses the posterior p(θ|x) onto specific concepts.

### Mapping to Prisms

| Bayesian Term | Project Term | Interpretation |
|---------------|-------------|----------------|
| Prior p(θ) | Vanilla model | Flat prior over analytical tasks → generic output |
| Posterior p(θ|prompt) | Prism-activated model | Prior collapsed onto specific analytical mode |
| Concept θ | Analytical operation | "Find conservation law", "name concealment", etc. |
| Prompt evidence | System prompt | Prism = strong evidence specifying which concept |

**Why prism is the dominant variable:** A well-specified prism provides strong evidence → tight posterior → low-variance, deep output. A vanilla prompt provides weak evidence → broad posterior → high-variance, shallow output. Model capacity affects the prior (richer models = richer mixture), but the prism collapses the posterior regardless.

### Novel Prediction: Output Variance

**If prisms are prior specifications, output VARIANCE under prisms should be lower than under vanilla at equivalent mean depth.**

The project measures mean depth but has never measured variance. This is a directly testable prediction:
- Run same prism on same target 5 times
- Run vanilla on same target 5 times
- Compare variance of depth scores

**Expected result:** Prism variance < vanilla variance (tighter posterior = more consistent outputs).

**Connection to Haiku pass@3:** The project already notes "Haiku pass@3: run 3x, expect 1 champion" — this implies high variance. If the Bayesian framework is correct, the variance should be LOWER for well-specified prisms (L12) than poorly specified ones (l12_universal on off-domain targets).

### Connection to "Code Nouns as Mode Triggers"

In the Bayesian framework: "this code's" is a strong likelihood function that activates the "code analysis" concept cluster. "this input's" is a weaker likelihood → broader posterior → more variable and shallower output. The vocabulary literally narrows the hypothesis space.

### Limitations

1. Xie's proof works for HMM mixtures — transformer pretraining distribution is much more complex
2. The "concept" = "analytical operation" mapping is a metaphor, not a formal equivalence
3. The Bayesian framework doesn't explain WHY construction bypasses the capacity gate (that's CLT's domain)

## Synthesis: Three Frameworks Converge

| Finding | CLT Explanation | Dual Process | Bayesian |
|---------|----------------|-------------|----------|
| L7→L8 threshold | Intrinsic load reduction | System 2→1 mode shift | N/A (doesn't explain) |
| 4-op sweet spot | Element interactivity limit | N/A | N/A |
| Compression floor | Minimum germane load | N/A | Minimum evidence for posterior collapse |
| Prism dominance | Schema activation | System 1 trigger | Prior specification |
| Code nouns | Reduce extraneous load | Mode switch | Strong likelihood function |
| Conservation laws | Schema output | Pattern completion output | Posterior mode |

**The frameworks are complementary, not competing:**
- **CLT** explains the CAPACITY constraints (why L7 fails, why 4 ops is the limit)
- **Dual Process** explains the MODE dynamics (why construction bypasses reasoning)
- **Bayesian** explains the MECHANISM (why the prism, not the model, is the dominant variable)

Together they predict: a well-designed prism (1) reduces element interactivity below the model's capacity threshold (CLT), (2) activates System 1-compatible procedural generation (dual process), and (3) collapses the posterior onto a specific analytical concept (Bayesian). The result is deep, consistent, capacity-independent analysis.
